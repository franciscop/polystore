Polystore

A universal library for standardizing KV-stores. Great when you don't want to depend on a specific implementation, or for library builders that want to accept any KV store available. 

npm install polystore

Familiar

It's a KV library. It has add, get, set, has and del. No dependencies, no surprises.

üß© Many different clients

We support 8+ clients, with a very clear documentation and easy way to add new clients.

You only need to define 3 methods to create a new client, .get(), .set() and .entries(). 

import kv from "polystore";

const store1 = kv(new Map());
const store2 = kv(localStorage);
const store3 = kv(redisClient);
const store4 = kv("cookie");
const store5 = kv("file:///users/me/data.json");
const store6 = kv(yourOwnStore);

üèñÔ∏è Clear and intuitive methods

A set of high-performance atomic operations with .add(), .set(), .get(), .has() or .del()

You can also work with all of entries with .keys(), .values(), .entries(), .all() and .clear(). 

import kv from "polystore";
const store = kv(new Map());

const key1 = await store.add("value2");
const key2 = await store.set("key1", "value1");
const isthere = await store.has("key1");

const allKeys = await store.keys();
const obj = await store.all();

üõó Operate on substores intuitively

We created the `.prefix()` to allow you to work with a subset of items intuitively.

e.g. if you have a session store and want to create a subset of sessions only for auth, you could do const auth = session.prefix('auth:'); and then use auth like if it was an independent, KV store. 

import kv from "polystore";
const store = kv(new Map());

const auth = store.prefix("auth:");
auth.set("key1", "value1");
console.log(await auth.entries());
// [["key1", "value1"]]

console.log(await store.entries());
// [["auth:key1", "value1"]]
